Use the craft-docs skill pattern. Here's the workflow you'd follow in a prompt:

 │ Prompt template:
 │
 │ ```
 │   Create documentation for [feature name]. Run the craft-docs skill.
 │
 │   Feature: [brief description]
 │   Slugs to create: [list of slugs]
 │   Usage: <FeatureHelp slug="[slug]" anchorId="[anchor]" />
 │ ```

 The skill automates:
 1. Create — drafts content/docs/[slug].mdx with title, description, tooltipSummary
 2. Anchor — adds heading anchors like ## My Heading {#my-heading}
 3. Output — gives you the exact JSX to paste
 4. Verify — runs npm run audit:docs to confirm everything is in sync

 So a full prompt would look like:

 │ "Document the education system selection feature. Create content under education-system slug. The tooltip should explain
 │ curriculum frameworks. Run craft-docs."
